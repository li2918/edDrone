Section 05 · Legal & Compliance
Risk Advisory · 不构成正式法律意见 · For Information Only

美国教育无人机最大的法律风险，不是"能不能飞"，而是身份、地点、数据、对象

从公开法规和官方指引看，美国教育无人机最核心的合规判断有四层：第一，飞行到底属于 recreational 还是 Part 107；第二，是否触发 FAA 的注册、Remote ID、空域授权和 FRIA 要求；第三，是否涉及学生隐私、照片、视频、地理位置和第三方平台数据；第四，学校、场地、州/地方政策与合同责任是否额外加码。

English Version → Domains · FAA + 学生隐私 + 州地方法规 + 合同责任 High-Risk Triggers · 室外 / 拍学生 / 商业化 / 第三方 SaaS
Misconception · 最易误判
教育用途 ≠ 自动豁免
FAA 明确指出，规则取决于飞行目的与操作方式；"教育"并不自动豁免。
Tool · 学校最关键合规工具
FRIA / Remote ID
教育机构可申请 FRIA；没有 Remote ID 设备的室外飞行，FRIA 是最现实合规抓手。
Data Risk · 课程机构最常见
FERPA + COPPA
一旦涉及学生记录、图像、声音、位置数据或儿童导向 app，隐私法和供应商合同条款成为门槛。
05.1

执行结论：先建立标准化合规判断表，再投入采购

如果你要在美国做教育无人机项目，最稳妥的理解方式不是"先买设备再说"，而是先建立一张标准化合规判断表。

Key Takeaway

合规分析顺序：身份 → 场景 → 数据。任何一步缺失，都可能让看似"教育用途"的项目落进 Part 107 或 FERPA / COPPA 的合规盲区。

Step 01 先定身份
先判断是 recreational 还是 Part 107，再决定测试、证照、注册和运营要求。
Step 02 再定场景
室内、室外、校园、机场附近、赛事场地，风险完全不同。
Step 03 最后定数据
一旦拍学生、存视频、记录位置、接第三方 app，法律风险会从航空法扩展到隐私和合同法。
Output 合规包
把以上三步固化成可复用的 Drone Compliance Pack，对销售、教学、家长、学校通用。
05.2

核心风险矩阵：9 类法律风险与对应执行动作

下面按管理层最关心的角度，整理出美国教育无人机项目最需要提前处理的法律与合规问题。

Key Takeaway

9 类风险中，FAA 合规是入场资格，FERPA / COPPA 决定能否进入学校采购。任一缺失都会让长周期销售在法务环节被卡住。

飞行身份认定 FAA Recreational Flyers、Part 107 美国无人机规则首先看飞行目的。为商业、机构服务或非纯娱乐目的飞行，通常不能简单按 recreational 处理。 可能错误适用 TRUST 而不是 Part 107 体系，导致证照和操作规则错误。 给每类课程、营地、学校项目建立书面分类标准，并由项目经理确认。
注册与 Remote ID FAA Registration、Remote ID、FRIA 多数需要注册或已注册的无人机必须符合 Remote ID；教育机构可申请 FRIA。 室外课程无法合法开展，或被迫限制设备型号。 优先选择标准 Remote ID 设备，或把校内室外项目设计在已获批 FRIA 框架内。
空域与地点 FAA Where Can I Fly、Airspace 101、No Drone Zone 靠近机场、特殊活动、TFR、政府设施附近，都可能触发额外限制。 活动当天无法飞行，或误入受限空域带来执法风险。 每次室外活动前做 B4UFLY / 空域检查，并在 SOP 中保留截图与审批记录。
州与地方规则 FAA State and Local Regulation Fact Sheet 虽然 FAA 主管空域，但地方政府和学校对起降地点、场地使用、校园规则仍可能有约束。 出现"联邦可飞，但校区/公园/场馆不让起降"的落地冲突。 把场地许可、校方书面批准和地方规则检查列为活动前置条件。
学生隐私与教育记录 FERPA 官方 FAQ 如果学校或供应商处理教育记录中的学生可识别信息，FERPA 约束会进入合同与运营流程。 学校采购被法务卡住，或后续数据共享触发违规风险。 供应商协议中写明 data use、direct control、redisclosure limits、retention policy。
儿童在线隐私 FTC COPPA FAQ 若 app 或在线服务面向 13 岁以下儿童，且收集照片、视频、声音、位置或 persistent identifiers，就可能触发 COPPA。 家长同意机制、广告设置、第三方 SDK、分析工具都可能出问题。 对面向低龄学生的平台做 COPPA 审查，默认关闭不必要追踪与精准位置。
图片、视频和地理位置元数据 FTC COPPA FAQ；FERPA 视具体记录而定 学生图像、声音和精确位置是高敏感区，尤其在展示、招生宣传和云端存储中。 市场部与教学团队可能在不知情下把合规问题扩大。 把 media release、metadata stripping、最小化存储做成默认机制。
室内 vs 室外 FAA FAQ: indoors only FAA 明确说明仅在室内进行的操作不适用 Part 107，但这不等于没有安全、合同和隐私责任。 团队容易误以为"室内就无需任何规则"，忽略伤害、录像和场地方责任。 把室内项目作为低航空风险场景，但保留校方许可、保险和安全 SOP。
商业宣传与对外承诺 FTC 一般广告与商业实践原则；合同风险为推断 如果对学校宣传"合规、安全、隐私保护"，这些表述会进入采购审查和合同承诺。 销售说辞与实际操作不一致，可能带来纠纷和信誉损害。 统一销售材料、DPA 模板、家长告知书和学校问答口径。

05.3

按场景看风险强度：低中风险 vs 中高风险

同样是"教育无人机"，不同业务形态的合规负担差距非常大。

Key Takeaway

"室内默认 + 室外审批"是最稳的运营默认值。商业化、对外宣传、社交媒体素材一旦触发，必须升级到中高风险流程。

低 — 中风险场景

  • 室内课堂演示，不上传学生数据，不做公开宣传。
  • 室内编程练习，设备不出楼宇，不使用带精准定位的在线平台。
  • 教师 PD 或成人培训，不涉及儿童数据收集。

中 — 高风险场景

  • 室外校内课程，尤其是靠近机场或受控空域。
  • 学校项目需要记录学生飞行视频、成绩、账户、地理位置或上传第三方平台。
  • 营地、公开活动、比赛、营销素材拍摄和社交媒体传播。
  • 代表学校或机构提供服务、收费教学、对外展示或品牌活动。
05.4

最容易踩坑的 6 个误区

下面这些误区在美国教育无人机场景中尤其常见。

Key Takeaway

所有 6 个误区的共同根源：把"教育"等同于"豁免"。在美国合规体系中，"教育"只是身份，不是免责证明。

误区 1

"教育用途天然豁免"

不成立。FAA 的规则判断更看飞行目的与操作情境，而不是简单看"是不是学校项目"。

误区 2

"室外操场肯定能飞"

不成立。操场上空可能仍受空域、校方政策、地方起降规则或临时限制影响。

误区 3

"学生视频只是教学素材，不算敏感数据"

风险很高。照片、视频、声音、位置数据都可能触发隐私和合同问题。

误区 4

"平台账户由供应商保管就行"

学校法务通常会追问谁控制数据、谁可再披露、保存多久、删除机制是什么。

误区 5

"室内就完全没有法律风险"

FAA 风险下降，但人身伤害、设备损坏、未成年人影像和场地方责任仍然存在。

误区 6

"先卖进去，后面再补合规"

教育采购里这通常会反过来拖慢成交，因为学校会在法务和采购环节集中补问。

05.5

建议执行清单：4 项高优先级动作

以下不是法律意见，而是基于公开规则整理出的高优先级执行清单。

Key Takeaway

合规不是销售障碍，而是销售加速器 —— 学校法务通过的速度，决定客户决策周期。提前准备好的 Compliance Pack 通常能把成交周期缩短 30%+。

Priority 1

做一份统一的 Drone Compliance Pack

至少包含：飞行分类标准、空域检查 SOP、Remote ID / FRIA 说明、家长同意模板、媒体授权、隐私问答、供应商数据处理说明。

Priority 2

把"室内默认化，室外审批化"

对于早期项目，优先采用室内课程降低 FAA 层面的不确定性；室外活动则进入审批和记录流程。

Priority 3

把数据最小化写进产品和运营

默认不收不必要位置数据，不开放不必要追踪，不长期保留学生视频，不把营销拍摄与教学数据混用。

Priority 4

学校合同必须预留隐私与责任条款

尤其是如果使用第三方 app、云平台、赛事平台或外包摄影，法务问答要提前准备。
